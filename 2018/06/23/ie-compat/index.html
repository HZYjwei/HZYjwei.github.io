ie-compat

发表于 2018-06-23 |

前言

首先点开ie8浏览器上面的工具——>Internet选项——>安全——>自定义级别——>(拉到最下面)其他——>通过域访问数据源——>启用
很重要啊,同源脚本的都不能加载

#
首先得认识

记得转码字符串吗

  • 小于 < ( less than ) lt;
  • 大于 > ( great than ) gt;
    这里也是一样 lt（小于）e(equal) 等于
    就是(小于等于IE 8) 的浏览器可以识别
    用于按需加载不同的文件，兼容必备

<HEAD>中

一般在HEAD中加载

respond.js

  • 1：将中的所有link引入的css文件都取出来放入数组，
  • 2：遍历数组一个个发送ajax请求
  • 3：ajax的callback分析reponse中的@media 和max-width，min-width分析不同viewprot下的css块
  • 4：window.onload和window.onrize的时候，获取document.body.offsetWidth + docuemnt.documentElement.offsetWidth然后将相应的css的class名赋到元素上。

加得越早，屏幕闪烁越快,用户体验越好（当你足够快时，队友的问号就跟不上你）

html5shiv.js & html5media.js

含有部分html5标签,讲道理没什么用
html5meida兼容了video和audio标签

尾部（body末尾加）

rem.js

rem.js
使用rem布局必备，rem这玩意好用得不得了
只要在判断userAgent然后按需加载就好了
https://github.com/chuckcarpenter/REM-unit-polyfill

jqu ery

需要2.0以下(hexo真毒，连打会解析不出)

CSS部分

首先就是.htc文件，这个其实就是.js包了层皮

ie-css3.htc

如果使用了text-shadow,box-shadow,border-radius
需要加上属性position:relative；
在你写的css后面加上behavior:url()；

这个最好不要多用，在一个文件中渲染有上限，如果写的过多，可能会导致后面的元素背景色和元素不在一起，有黑色的条（但是审查不到这个条元素是什么）之类的

backgroundsize.htc

兼容了background-size：contain | cover
在写css的时候最好不要写复合属性如

当一个识别不出来好像整个样式都不能识别

CSS

ie7

width

在做向下三角的时候不设置width会默认有个width；

display:inline-block

在ie7中

如果原来是display：block（div）中设置display：inline-block 那么这个元素的下一个会换行

float：right：

同样是换行问题
建议改为display：absolute；

position层级问题

;
要在ie7中topBar的userMenu覆盖contianer，
必须在topBar和container中都加入position：relative，而且z-index大于被覆盖元素

:after&:before

我一直用::after和::before来清除float的浮动流
但是ie8中只支持:after形式的
而ie7貌似不支持伪元素写法
